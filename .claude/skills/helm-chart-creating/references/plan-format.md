# 計画書フォーマット

Phase 2 で生成する計画書の厳密フォーマット。本ファイルから逸脱しない。

## 大原則

- **計画書は確定済み内容のみ載せる** (Phase 1 で全て擦り合わせ済み)
- **設定ファイル中身を計画書で初出ししない** (Phase 1 で確定済みのものを載せる)
- **「後で決める」 「未設定」 のままにしない** (能動的提案の責務)
- **保留項目** (ユーザーが今提供できない情報) は冒頭に明示

---

## 計画書構造

### 1. 基本情報

| 項目 | 値 |
|---|---|
| チャート名 | xxx |
| ワークロード種別 | xxx |
| 対象 k8s バージョン | xxx |
| 出力先ディレクトリ | xxx |
| 作成日時 | yyyyMMddHHmm |

### 2. 保留項目 (該当する場合のみ)

ユーザーが現時点で提供できない情報 (`image.digest` の決定待ち等) を列挙する。Phase 3 の chart 生成時は空欄で生成し、ユーザーが後日埋める。

### 3. ⚠️ 未設定パラメータ警告 (優先度付き)

デフォルト値のままにしているパラメータのうち、ユーザーが意識すべきものを列挙する。**優先度を付けて分類する**:

#### 🔴 強く推奨 (アプリ特性から判断して設定すべき)

ベストプラクティスから判断して、このアプリ特性ではほぼ必ず設定すべきもの。Claude は Phase 1 で能動提案したが、ユーザーが「今は不要」 「後で決める」 と判断したもの。

| パラメータ | 現在値 | Claude の推奨 | ユーザーの判断理由 |
|---|---|---|---|
| ... | ... | ... | ... |

**このセクションは可能な限り空にすることを目指す**。Phase 1 の能動提案で確定するのが理想。

#### 🟡 環境次第 (要件があれば設定)

組織や運用要件によって設定するかが変わるもの。Claude では判断できないため、ユーザーに最終確認を促す。

| パラメータ | 現在値 | このような場合は設定が必要 |
|---|---|---|
| ... | ... | ... |

#### ⚪ 通常デフォルト OK

参考情報。通常はデフォルトのままで問題ない項目。短く列挙のみ。

---

### 4. ヒアリング結果サマリ

ユーザー要望と確定事項を 3-5 行で要約。

### 5. 決定方法サマリ

| 決定方法 | 件数 |
|---|---|
| ユーザー指定 | N 件 |
| ベストプラクティス能動提案 (Phase 1 でユーザー承認済み) | N 件 |
| 組織標準 (organization-standards.md) | N 件 |
| デフォルト値そのまま | N 件 |

### 6. パラメータ詳細

カテゴリ別 (image / ServiceAccount / セキュリティ / リソース / Service / health / scheduling / 拡張ポイント / ワークロード固有) のテーブル。各行に:

| パラメータ | 値 | 挙動 | 決定方法 |
|---|---|---|---|

「決定方法」 列は以下のいずれか:
- `ユーザー指定`
- `Phase 1 能動提案 (ユーザー承認済み)`
- `組織標準` (`organization-standards.md`)
- `デフォルト`

### 7. 設定ファイル (該当する場合のみ)

Phase 1 で確定した設定ファイル中身 (rsyslog.conf 等)。**ここで初出する内容は禁止**。Phase 1 ヒアリング時にユーザーと擦り合わせ済みのもののみ。

```
# 設定ファイル名
(中身)
```

「Phase 1 で確認した前提」 (例: rsyslog v8 系、RainerScript 構文) を併記する。

### 8. 環境別 values 設定

| 環境 | 差分 |
|---|---|
| dev | image.digest, replicaCount=1 |
| stg | image.digest, replicaCount=2 |
| prod | image.digest, replicaCount=3 |

### 9. 生成されるリソース一覧

このチャートで作成される k8s リソースを列挙 (Deployment, Service, ConfigMap 等)。

### 10. セルフチェック結果

Claude が Phase 2 計画生成時に行ったセルフチェックの結果を明示:

| チェック項目 | 結果 |
|---|---|
| hostPort と Service.NodePort の混同なし | ✅ |
| HPA 有効時に resources.requests.cpu 設定済み | ✅ |
| replicaCount=1 で PDB minAvailable=1 になっていない | ✅ |
| readOnlyRootFilesystem=true 下で書き込み領域 (emptyDir) 確保済み | ✅ |
| バージョン依存パラメータが対象 k8s バージョンで利用可能 | ✅ (web_search で確認済み) |
| ... | ✅ |

各 behavior-pattern の「よくある落とし穴」 セクションをチェック対象とする。

### 11. 確認事項

```
この計画で進めてよろしいですか?

★ 特に「3. 未設定パラメータ警告」 セクション、特に 🔴 強く推奨 を確認してください。

意図に合致していれば「OK」 「進めて」 等で承認してください。
修正が必要な場合は具体的に指摘してください。
```
