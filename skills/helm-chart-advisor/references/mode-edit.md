# 編集モード

既存の Helm chart を修正する場合の手順。

## ヒアリング

作業を始める前に、以下の情報がユーザーの依頼から読み取れるか確認する。不足している項目があれば、作業に着手せず先に確認する。回答の中で新たな不明点が出てきたら追加で確認してよい。

**必須（不明なら必ず確認）:**
- **対象ファイル**: どのチャートのどのファイルを変更するか（パスが明確でなければ確認）
- **変更内容**: 何をどう変えたいか（追加・修正・削除）

**曖昧になりやすいポイント（依頼内容から判断できなければ確認）:**
- **変更スコープ**: 依頼された箇所だけを修正するか、関連するベストプラクティス違反（命名規則、セキュリティ設定等）も合わせて修正するか
- **既存パターンとの整合性**: 既存チャートが独自の構造やパターンを持っている場合、それに合わせるかベストプラクティスに寄せるか

## 変更内容の確認

`confirmation-rules.md` のルールに従い、変更内容の確認をユーザーに提示する。編集では変更箇所と影響範囲を対象に行う。加えて以下も含める：

- **ベストプラクティス違反への対応**（変更スコープの確認時に「合わせて修正する」となった場合）: 追加で修正する項目とその理由

変更が軽微で影響が自明な場合（コメント追加、タイポ修正等）はこのステップを省略してよい。

## 手順

1. **ヒアリング**: 上記のヒアリング項目に基づき、不足情報を確認する。レビュー結果ファイル（`result/helm-chart-advisor-review-*.md`）が引数として渡された場合は、その内容を変更仕様として読み込む
2. まず既存のファイル構成を確認し、`chart-structure.md` のスキャフォールド標準構造と比較する（Chart.yaml, values.yaml, templates/ 配下に加え、.helmignore, _helpers.tpl, NOTES.txt, serviceaccount.yaml, tests/ の有無も確認する）。不足ファイルがあれば変更スコープの確認時にベストプラクティス違反として報告し、合わせて追加するか確認する
3. **変更内容の確認**: 上記の確認に基づき、変更箇所の挙動説明・不整合チェックを行い、ユーザーの OK を得る（軽微な変更は省略可）
4. **プラン出力**: 確認済みの変更内容を `result/helm-chart-advisor-edit-plan-{yyyyMMddHHmm}.md` に保存する。`assets/edit-plan-template.md` のテンプレートに従うこと（軽微な変更で確認を省略した場合はこのステップも省略可）
5. **プランに基づいて変更**: 手順4で出力したプランファイルをインプットとして、以降の作業を行う。変更箇所に関連するリファレンスを参照し、ベストプラクティスを適用する
6. 既存の values 構造やテンプレートパターンとの一貫性を維持する
7. 変更後に `helm lint` と `helm template` で検証する

## 変更箇所とリファレンスの対応

| 変更対象 | 参照ファイル |
|---|---|
| Chart.yaml（バージョン、メタデータ） | `chart-structure.md` |
| values.yaml（パラメータ追加・変更） | `values.md` |
| テンプレート（構造、フォーマット） | `templates.md` |
| 依存チャート、CRD | `dependencies.md` |
| securityContext、NetworkPolicy、ServiceAccount | `security.md` |
| イメージ、リソース、プローブ、ラベル | `workloads.md` |
| デプロイ運用、CI/CD、テスト、シークレット | `operations.md` |
| OCI、Helmfile、ArgoCD、ドキュメント | `ecosystem.md` |

変更箇所に関連するファイルだけを読むこと。全ファイルを一括で読み込まない。
